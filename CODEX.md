# Codex 工作规则

## 佛脚刷题入口规则

1. 佛脚刷题入口使用 `data/banks.json` 维护。
2. 佛脚刷题页面为 `pages/banks.html`，不按课程分类，使用搜索框检索。
3. 二维码图片统一放入 `images/banks/`。
4. 不得上传或公开题目原文、老师内部题库文件、学校敏感资料、同学个人信息。
5. 只允许公开展示佛脚码、二维码、课程、题库标题、状态、更新时间、来源和备注。
6. 单个题库使用普通记录；分章节题库使用 `kind: "collection"` 和 `items`。
7. 修改 `data/banks.json` 后必须运行 `scripts/validate_banks.py`。

本仓库用于医学本科课程复习资料生成与发布。所有自动化、整理、生成和发布任务都必须遵守以下规则。

## 基本定位

1. 本仓库用于医学本科课程复习资料生成与发布。
2. 默认使用中文。
3. 正式输出资料必须面向复习、背诵、考试理解。
4. 不得编造医学事实。
5. 对来源材料没有明确支持的内容，必须标记为“待核实”或 TODO。
6. 不得上传患者隐私、同学个人信息、学校内部敏感资料。
7. 原始 PPT/PDF/Word 不默认上传 GitHub，除非用户明确要求。
8. 下载站和正式输出资料必须包含免责声明：资料仅用于课程学习与考试复习，不作为临床诊疗依据；待核实内容需以教材、指南或授课材料为准。

## 文件输出规则

1. 已确认发布的文件统一放入 `files/课程目录/`。
2. 普通生成完成后，不得自动放入 `review/`。
3. 只有用户明确要求上传、发布、放入 `review/` 或生成发布草案时，才可以将成品复制到 `review/课程目录/`。
4. 待发布成品进入 `review/` 时，必须同时生成 `*.publish-draft.json` 给用户检查。
5. 发布草案必须包含 `metadata`、`review_path`、`website_url`、`download_url_after_publish` 和 `needs_user_review`。
6. 用户明确要求发布后，才将文件移动到 `files/课程目录/`。
7. 每次发布文件后，必须更新 `data/files.json`。
8. 每次发布文件后，必须更新 `tasks/task-log.md`。
4. 文件命名使用日期 + 英文短名 + 类型，例如：

```text
2026-06-17-acute-osteomyelitis-review.pdf
```

9. 网页 `index.html` 应读取或展示 `data/files.json` 中的文件列表。
10. 每份正式资料应在首页、封面、页脚或文末保留免责声明。
11. 当前默认课程目录固定为 `imaging`、`diagnostics`、`preventive-medicine`、`pharmacoeconomics`、`surgery`、`others`。
12. 系统必须保持可扩展；新增课程时，可新增 `files/新课程目录/`，并在 `data/files.json` 中使用新的 `category`。
13. 下载站只作为成品文件仓库，不预设每次必须生成的文件格式；具体输出 PDF、Word、PPT、HTML 或其他格式，由单次任务策略决定。
14. 同一份笔记的多个格式版本应使用相同的 `note_id`，网页应合并为一条笔记并显示多个下载选项。
15. 首页资料卡片的核心信息为名称、 一句话简介、下载方式；`description` 应尽量控制为一句话。
16. 下载站按课程分区展示资料，不设置全站搜索；每个课程区块保留本课程内部搜索。
17. 发布前必须运行 `scripts/pre_publish_check.py` 和 `scripts/validate_files.py`。
18. `data/files.json` 字段规范以 `docs/files-json-fields.md` 为准。
19. 用户确认发布 `review/` 中的文件时，优先使用 `scripts/publish_review_file.py --metadata` 根据发布草案完成移动、索引更新和任务日志记录。
20. HTML 在线预览文件统一放入 `notes/课程目录/`，首页只通过 `previewUrl` 链接到预览页，不直接嵌入完整 HTML 内容。
21. 下载按钮优先使用 `downloadUrl`；未填写时使用 `path`。
22. 如果资料本身是另一个 GitHub Pages 学习站点，可以作为课程索引里的 HTML 条目维护：填写 `course`、`category`、`type: "HTML"` 和外部 `previewUrl`，不需要 `path` 或 `downloadUrl`，首页只显示“在线预览”入口。
23. `admin.html` 是半自动索引编辑器，只能导出新的 JSON 草稿；不得把它当作安全后台。用户通过它导出的 JSON 必须由 Codex 检查后，才能替换 `data/files.json`、运行脚本、提交和推送。

## 来源与医学准确性

1. 优先依据用户提供的原始资料。
2. 若需要补充医学背景知识，必须区分“来源资料内容”和“补充说明”。
3. 对未被来源材料明确支持的考点、数值、诊断标准、治疗原则，必须标记为“待核实”或 TODO。
4. 不输出未经核实的临床建议。
5. 复习资料应用于学习和考试复习，不替代专业医疗判断。
6. 公开下载页可以显示来源类型，例如“本地 PPT 整理”“教材整理”“错题整理”，但不得显示原始文件名、本地路径或内部资料编号。

## 隐私与上传边界

1. 不上传患者隐私、病例截图、检查号、病历号、身份证号、手机号、住址、头像或其他可识别个人的信息。
2. 不上传同学个人信息、班级内部名单、联系方式、成绩、聊天记录等。
3. 不上传学校内部敏感资料。
4. 原始 PPT、PDF、Word 默认保存在本地，不上传 GitHub。
5. 如果用户明确要求上传原始资料，必须先提醒检查隐私和版权风险。
6. 公开仓库只发布整理后的系统性学习笔记，不发布原始课件或原始资料。

## 每次任务完成后的回复摘要

每次修改完成后，必须输出修改摘要，包括：

- 新增了哪些文件
- 修改了哪些文件
- 生成文件路径
- 下载网页是否已更新
- 是否需要用户人工检查
